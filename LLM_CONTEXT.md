# 🤖 LLM 컨텍스트 가이드

> 이 문서는 LLM(AI 어시스턴트)이 효과적으로 문제를 추천하고 학습을 지원하기 위한 가이드입니다.

---

## 📁 프로젝트 개요

**목적**: 코딩테스트 실력 향상을 위한 체계적인 문제 풀이 및 아카이빙
**현재 상태**: 중급 레벨 (약 1,165문제 풀이 완료)
**주 언어**: C++ (C++20 표준)

### 핵심 문서 구조

| 파일명 | 용도 | LLM 활용 방법 |
|--------|------|---------------|
| **README.md** | 프로젝트 소개, 사용법 | 초기 설정 및 워크플로우 이해 |
| **SOLVING_LOG.md** | 날짜별 문제 풀이 기록 | 최근 학습 패턴, 진행 상황 파악 |
| **SKILL_ANALYSIS.md** | 알고리즘별 강/약점 분석 | 약점 파악, 학습 우선순위 결정 |
| **LLM_CONTEXT.md** (이 문서) | 문제 추천 로직, 학습 전략 | 추천 알고리즘 및 규칙 |

### 디렉토리 구조
```
COTE/

```

## 🎯 문제 추천 로직

### 📌 기본 원칙

#### 1. **다양성 우선, 약점 보완 병행**
- ❌ **하지 말 것**: 같은 알고리즘만 연속 3개 이상 추천
- ✅ **해야 할 것**: 2~3개 알고리즘을 섞어서 추천
- ✅ **예외**: 사용자가 특정 알고리즘에서 어려움을 겪으면 해당 알고리즘 집중 추천

**예시**:
```
✅ 좋은 추천 순서: DP → 이분탐색 → 백트래킹 → DP → 그리디
❌ 나쁜 추천 순서: DP → DP → DP → DP → DP
```

#### 2. **적응형 난이도 조절**

| 상황 | 판단 기준 | 조치 |
|------|-----------|------|
| **너무 어려움** | 힌트 2회 이상 요청 / 30분+ 소요 / 포기 표현 | 난이도 **1단계 하락** |
| **적정 수준** | 힌트 0~1회 / 스스로 해결 / 긍정적 피드백 | 난이도 **유지** |
| **너무 쉬움** | 5분 이내 해결 / "쉽다" 표현 | 난이도 **1단계 상승** |

**난이도 단계** (백준 기준):
```
브론즈 5~3 → 브론즈 2~1 → 실버 5~4 → 실버 3~2 → 실버 1 → 골드 5~4 → ...
```

#### 3. **우선순위 기반 추천**

**추천 순서** (높음 → 낮음):
1. **최우선 약점** (문제 수 < 5개): DP, 이분탐색, 백트래킹
2. **중요 약점** (문제 수 < 10개): 자료구조, 트리, 다익스트라, 비트마스킹
3. **보완 필요** (문제 수 10~15개): 해시, 기하학, 두포인터
4. **강점 유지** (문제 수 20개+): 가끔 섞어서 자신감 회복용

### 📊 추천 알고리즘 결정 프로세스

```
1. SOLVING_LOG.md에서 최근 3개 문제 확인
   ↓
2. 같은 알고리즘이 2개 이상?
   → YES: 다른 알고리즘 추천
   → NO: 3단계로
   ↓
3. SKILL_ANALYSIS.md에서 약점 알고리즘 확인
   ↓
4. 문제 수가 가장 적은 상위 3개 중 하나 선택
   ↓
5. 최근 풀이 난이도 기준으로 비슷한 난이도 추천
   ↓
6. 사용자에게 추천 (1개씩만!)
```

## 📝 문제 추천 템플릿

문제를 추천할 때 다음 형식을 사용하세요:

```markdown
## 추천 문제

**백준 [문제번호]번 - [문제명]** ([난이도], [알고리즘])

### 추천 이유:
1. **[주요 이유]**: [구체적 설명]
2. **난이도**: [현재 수준과 비교]
3. **학습 가치**: [이 문제로 얻을 수 있는 것]

### 문제 개요:
[1~2문장으로 문제 설명]

[격려 메시지]
```

**예시**:
```markdown
## 추천 문제

**백준 11053번 - 가장 긴 증가하는 부분 수열** (실버2, DP)

### 추천 이유:
1. **DP 심화 학습**: 1463번 이후 LIS 패턴 학습에 최적
2. **난이도**: 실버2로 적절한 도전
3. **학습 가치**: 2D DP와 이분탐색 최적화까지 확장 가능

### 문제 개요:
수열에서 증가하는 부분 수열 중 가장 긴 것의 길이를 구하는 DP 대표 문제

다음 단계로 나아갈 준비가 되었습니다! 🚀
```

## 🔄 학습 상황별 대응 전략

### Case 1: 연속 성공 (3문제 이상)
- **판단**: 현재 난이도가 적절하거나 쉬움
- **조치**: 난이도 1단계 상승 또는 새로운 알고리즘 도입
- **예시**: "실버4 연속 해결 → 실버3 추천"

### Case 2: 연속 실패/어려움 (2문제 이상)
- **판단**: 개념 이해 부족 또는 난이도 과다
- **조치**: 같은 알고리즘의 더 쉬운 문제 추천
- **예시**: "실버3 어려워함 → 실버4~5 기초 문제 추천"

### Case 3: 알고리즘 편식
- **판단**: 최근 5문제 중 3개 이상이 같은 알고리즘
- **조치**: 강제로 다른 알고리즘 추천
- **예시**: "DP 3연속 → 이분탐색 또는 백트래킹 추천"

### Case 4: 처음 보는 알고리즘
- **조치**:
  1. 개념 간단 설명 제공
  2. 가장 쉬운 대표 문제 추천
  3. 풀이 후 개념 정리 유도
- **예시**: "비트마스킹 처음 → 개념 설명 + 쉬운 예제 문제"

- [ ] 공간 복잡도가 적절한가?

- **백준**: https://www.acmicpc.net/problem/[문제번호]
- **프로그래머스**: https://programmers.co.kr/

### 난이도 가이드 (백준)
- **브론즈**: 기초 구현, 입출력
- **실버**: 기본 알고리즘 (DP, DFS/BFS, 이분탐색 등)
- **골드**: 심화 알고리즘, 복합 문제
- **플래티넘+**: 고급 알고리즘, 최적화

### 알고리즘 난이도 순서 (쉬움 → 어려움)
```
구현, 문자열, 정렬
→ DFS/BFS, 그리디, 브루트포스
→ DP, 이분탐색, 백트래킹
→ 다익스트라, 플로이드-워셜, 위상정렬
→ 세그먼트 트리, 비트마스킹, 트라이
```
