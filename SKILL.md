## 💻 코딩 스타일 분석

### ✅ 좋은 습관

#### 1. 주석 작성이 매우 자세함
```cpp
// 바텀업 방식으로 구현하기
// D[i] : i에서 1로 만드는 데 걸리는 최소 연산 횟수
// D[i] = D[i-1] + 1
```
- **장점:** 나중에 코드를 다시 봐도 이해하기 쉬움
- **유지:** 계속 이렇게 작성하세요!

#### 2. 빠른 입출력 최적화
```cpp
ios::sync_with_stdio(false);
cin.tie(NULL);
cout.tie(NULL);
```
- **장점:** 시간 제한이 빡빡한 문제에서 유리
- **유지:** 좋은 습관입니다

#### 3. STL 활용 능력
```cpp
vector<vector<int>> Bat;
unordered_set<int> Su1;
```
- **장점:** 적절한 자료구조 선택
- **유지:** 다양한 STL 계속 활용하세요

#### 4. 방향 배열 활용
```cpp
static int dx[4] = { 0, 0, -1, 1 };
static int dy[4] = { -1, 1, 0, 0 };
```
- **장점:** 상하좌우 탐색이 깔끔함
- **유지:** 그래프 문제에 필수 패턴

### ⚠️ 개선이 필요한 습관

#### 1. 변수명 규칙 (중요도: 높음)
**현재:**
```cpp
static vector<vector<int>> Bat;   // 대문자 시작
static vector<vector<bool>> Visited;
int Temp;
```

**문제점:**
- C++에서 일반적으로 변수는 소문자로 시작 (camelCase 또는 snake_case)
- 대문자는 보통 클래스명이나 상수에 사용

**개선안:**
```cpp
// camelCase (권장)
static vector<vector<int>> bat;
static vector<vector<bool>> visited;
int temp;

// 또는 snake_case
static vector<vector<int>> bat_field;
static vector<vector<bool>> is_visited;
int temp_value;
```

**왜 중요한가:**
- 협업 시 팀 코딩 컨벤션에 맞춰야 함
- 읽기 쉬운 코드 = 버그 적은 코드
- 기업 코딩테스트에서 코드 가독성도 평가 요소

#### 2. VLA 사용 자제 (중요도: 중간)
**현재:**
```cpp
int n, k;
cin >> n >> k;
int coins[n];  // Variable Length Array (VLA)
```

**문제점:**
- C++ 표준이 아님 (C99 확장 기능)
- 일부 컴파일러에서 에러 발생 가능
- 스택 오버플로우 위험

**개선안:**
```cpp
int n, k;
cin >> n >> k;
vector<int> coins(n);  // vector 사용 (권장)

// 또는 크기가 고정이면
int coins[1000];  // 충분히 큰 고정 배열
```

#### 3. static 전역 변수 최소화 (중요도: 낮음)
**현재:**
```cpp
static int N;
static vector<int> D;
```

**문제점:**
- 여러 테스트 케이스 실행 시 초기화 필요
- 함수형 프로그래밍과 맞지 않음

**개선안:**
```cpp
// main 함수 내부에 선언하거나
int main() {
    int n;
    vector<int> d;
    // ...
}

// 함수 파라미터로 전달
void dfs(int x, int y, vector<vector<int>>& bat) {
    // ...
}
```

**단, 백준에서는:**
- 단일 테스트 케이스면 static 사용해도 무방
- 편의성이 더 중요할 수 있음
- 실전에서는 상황에 맞게 선택

#### 4. nullptr vs NULL (중요도: 낮음)
**현재:**
```cpp
cin.tie(NULL);    // 여기는 NULL
cin.tie(nullptr); // 여기는 nullptr
```

**권장:**
```cpp
// C++11 이후는 nullptr로 통일
cin.tie(nullptr);
cout.tie(nullptr);
```

### 🔥 최우선 (1-2개월)

#### 1. 다이나믹 프로그래밍 (DP) 집중 공략
**현재 상태:** 10개 → **목표:** 50개 이상

**추천 학습 순서:**
1. **기초 DP**

2. **중급 DP**
   - LIS (최장 증가 부분 수열)
   - 구간 DP
   - 비트마스킹 DP (5문제)

3. **고급 DP**
   - 트리 DP
   - 확률 DP

**추천 문제:**

- 백준 CLASS 3, 4의 DP 문제

#### 2. 이분탐색 (Binary Search)
**현재 상태:** 3개 → **목표:** 20개 이상

**학습 포인트:**

- 실수 이분탐색

**추천 문제:**
- 1920 (수 찾기), 2805 (나무 자르기)

#### 3. 백트래킹 (Backtracking)

**학습 포인트:**
